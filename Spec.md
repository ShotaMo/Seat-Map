# 座席表作成アプリケーション - 要件定義書

## 1. 概要
会場内のテーブルと座席を視覚的に配置し、各座席にメンバーを割り当てることができるWebアプリケーション。

## 2. 目的
- イベントや会議の座席配置を簡単に作成・管理できるようにする
- 視覚的なインターフェースで直感的に座席を配置できる
- メンバー情報（名前、X ID）を座席に紐付けて管理できる

## 3. 機能要件

### 3.1 会場管理
- **FR-1.1**: アプリケーション起動時に会場（Venue）が1つ表示される
- **FR-1.2**: 会場は座席配置のキャンバスとして機能する

### 3.2 テーブル管理
- **FR-2.1**: 会場内にテーブルを配置できる
- **FR-2.2**: テーブルの配置数は0～6個まで可能
- **FR-2.3**: テーブルを追加するボタンを提供する
- **FR-2.4**: テーブルを削除できる機能を提供する
- **FR-2.5**: テーブルをドラッグ＆ドロップで移動できる（オプション）

### 3.3 椅子管理
- **FR-3.1**: 各テーブルに椅子を配置できる
- **FR-3.2**: 1つのテーブルに配置できる椅子の数は0～8個まで
- **FR-3.3**: テーブルごとに椅子を追加するボタンを提供する
- **FR-3.4**: 椅子を削除できる機能を提供する
- **FR-3.5**: 椅子はテーブルの周囲に配置される

### 3.4 メンバー管理
- **FR-4.1**: 各椅子にメンバーを割り当てることができる
- **FR-4.2**: メンバー情報として以下を登録できる:
  - 名前（必須）
  - X（Twitter）のID（オプション）
- **FR-4.3**: 椅子をクリックするとメンバー情報入力フォームが表示される
- **FR-4.4**: メンバー情報を編集できる
- **FR-4.5**: メンバー情報を削除（クリア）できる

## 4. 非機能要件

### 4.1 ユーザビリティ
- **NFR-1.1**: 直感的なUIで操作が簡単であること
- **NFR-1.2**: レスポンシブデザインでモバイルでも閲覧可能であること

### 4.2 パフォーマンス
- **NFR-2.1**: ページ読み込み時間は3秒以内であること
- **NFR-2.2**: ユーザー操作に対する反応は即座（100ms以内）であること

### 4.3 互換性
- **NFR-3.1**: モダンブラウザ（Chrome、Firefox、Safari、Edge）の最新版で動作すること

### 4.4 保守性
- **NFR-4.1**: TypeScriptで型安全な実装を行うこと
- **NFR-4.2**: コンポーネント単位でテストが書けること

## 5. 制約事項
- テーブル数の上限: 6個
- 1テーブルあたりの椅子数の上限: 8個
- データの永続化は初期バージョンでは不要（将来の拡張として検討）

+--------------------------------------------------+

+--------------------------------------------------+
|                                                  |

|  +-------------+      +-------------+            |
|  | テーブル 1   |      | テーブル 2   |            |
|  | [削除]      |      | [削除]      |            |
|  | [椅子追加]   |      | [椅子追加]   |            |

|  | [座席][座席] |      | [座席][座席] |            |

|  +-------------+      +-------------+            |

### 6.2 メンバー情報入力フォーム
椅子をクリックすると表示されるモーダルまたはサイドパネル:
```
+---------------------------+
| メンバー情報               |
+---------------------------+
| 名前: [____________]      |
| X ID: [____________]      |
|                           |
| [保存] [キャンセル]        |
+---------------------------+
```

## 7. 用語定義
- **会場（Venue）**: 座席配置全体を表すコンテナ
- **テーブル（Table）**: 複数の椅子をグループ化する単位
- **椅子（Chair/Seat）**: メンバーが座る個別の座席
- **メンバー（Member）**: 座席に割り当てられる人物情報
